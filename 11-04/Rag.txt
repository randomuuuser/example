


- `RagTask` ‚Äî retrieves relevant passages from a knowledge base via
  a `RetrievalTask` component and generates an answer conditioned
  on them. *Ex: chatbot answering from internal documentation.*
  - `retrieval_task: RetrievalTask` ‚Äî retrieval component responsible
    for fetching relevant passages; replaces the former opaque
    `retriever: Any` and `vector_store: Any` attributes.
  - `generate(query) str` ‚Äî overrides `NLPGenerativeTask.generate()`;
    internally calls `self.retrieval_task.retrieve(query)` to fetch
    relevant passages, then generates an answer conditioned on them.


- `RetrievalTask` ‚Äî retrieves the most relevant passages from an
  indexed knowledge base given a query.
  - `embedding_task: EmbeddingTask` ‚Äî encodes the query into a dense
    vector for similarity search against the `vector_store`; ensures
    the same embedding model is used for both indexing and retrieval.
  - `vector_store: Any` ‚Äî indexed passage store queried at inference
    time (e.g. FAISS, Chroma, Elasticsearch).
  - `top_k: int` ‚Äî number of passages to retrieve per query.
  - `retrieve(query) List[str]` ‚Äî encodes the query via
    `self.embedding_task`, searches `self.vector_store`, and returns
    the top-k most relevant passages.


 RetrievalTask {
    +EmbeddingTask embedding_task
    +Any vector_store
    +int top_k
    +retrieve(query) List~str~
}

- `RerankingTask` ‚Äî reorders a list of retrieved passages by
  relevance to a query using a cross-encoder model. Typically
  used as a second-stage refinement step after `RetrievalTask`
  in a RAG pipeline.
  *Ex: given the query "What are the early redemption conditions?"
  and 10 retrieved contract passages, reorder them so the most
  relevant appear first before passing them to `RagTask`.*
  - `top_k: int` ‚Äî number of top-ranked passages to return after
    reranking.
  - `rerank(query, passages) List[str]` ‚Äî scores each
    (query, passage) pair using a cross-encoder and returns the
    top-k passages sorted by decreasing relevance score.


- `RagTask` ‚Äî retrieves relevant passages from a knowledge base via
  a `RetrievalTask` component, optionally reranks them via a
  `RerankingTask` component, and generates an answer conditioned
  on them.
  - `retrieval_task: RetrievalTask` ‚Äî retrieval component responsible
    for fetching relevant passages.
  - `reranking_task: Optional[RerankingTask]` ‚Äî optional reranking
    component; when provided, reorders retrieved passages by
    relevance before generation.
  - `generate(query) str` ‚Äî overrides `NLPGenerativeTask.generate()`;
    calls `self.retrieval_task.retrieve(query)`, optionally reranks
    via `self.reranking_task.rerank()`, then generates an answer
    conditioned on the final passage list.


class RerankingTask {
    +int top_k
    +rerank(query, passages) List~str~
}

NLPTask <|-- RerankingTask

class RagTask {
    +RetrievalTask retrieval_task
    +Optional~RerankingTask~ reranking_task
    +generate(query) str
}
